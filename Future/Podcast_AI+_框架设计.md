# AI+ 播客节目框架设计

---

## 一、节目定位

### 节目名称候选

| 名称 | 调性 | 适合风格 |
|------|------|----------|
| **AI+ 无限局** | 开放、跨界 | 轻松对谈，跨行业碰撞 |
| **涌现 Emergence** | 科技感、深度 | 深度思考，前沿探讨 |
| **下一步 Next Move** | 行动导向 | 实操、趋势判断 |
| **交叉点 Intersection** | 跨界、融合 | 行业交叉，多元视角 |
| **新变量** | 新锐、洞察 | AI作为各行业的"新变量" |

### 一句话定位

> 一档关于AI如何重塑世界、行业和个人的对话节目。
> 既聊AI前沿本身，也聊AI与每个行业碰撞时产生的真实变化。

### 目标听众

- **核心圈**：对AI感兴趣的科技从业者、创业者、产品经理
- **扩展圈**：各行业中正在被AI影响的专业人士（医疗、教育、金融、法律、设计等）
- **外围圈**：对未来趋势和世界局势感兴趣的知识型听众

### 节目调性

- 有深度但不学术——用对话拉近前沿和日常的距离
- 有观点但不偏激——鼓励多元视角，承认不确定性
- 有温度但不鸡汤——关注技术背后真实的人和故事

## 二、内容三大支柱

```
            ┌─────────────────┐
            │    AI+ 播客      │
            └────────┬────────┘
                     │
       ┌─────────────┼─────────────┐
       │             │             │
  ┌────▼────┐  ┌─────▼─────┐  ┌───▼─────┐
  │ 支柱 A  │  │  支柱 B   │  │ 支柱 C  │
  │ AI 前沿 │  │ AI × 行业 │  │ AI 与世界│
  │ 技术本身 │  │ 跨界对话  │  │ 宏观格局 │
  └─────────┘  └───────────┘  └─────────┘
```

### 支柱 A：AI 前沿（技术与产品）

聊AI本身的演进——模型、产品、工具链、开源生态。

**选题方向举例：**
- 大模型军备竞赛：Gemini vs Claude vs GPT vs DeepSeek，谁在赢？
- Agent 时代到来：从工具到数字同事的跨越
- 开源 vs 闭源：DeepSeek 现象与AI民主化
- AI 编程的真实体验：Cursor、Copilot、Windsurf 深度对比
- 百万 token 上下文意味着什么？
- AI 安全与对齐：我们应该担心什么？

### 支柱 B：AI × 行业（跨界碰撞）

邀请各行业从业者，聊AI在他们领域的真实落地。

**选题方向举例：**
- AI × 医疗：诊断准确率超人类之后会发生什么
- AI × 教育：老师会被取代还是被增强
- AI × 法律：AI律师助手的边界在哪里
- AI × 金融：量化、风控、客服的AI革命
- AI × 设计/创意：当AI能画能写，创意人何去何从
- AI × 制造业：智能工厂的理想与现实
- AI × 自媒体/内容：一个人的媒体帝国成为可能？
- AI × 农业/生物：冷门但深刻的变革

### 支柱 C：AI 与世界（宏观格局）

把AI放在更大的坐标系里——地缘政治、经济、社会。

**选题方向举例：**
- 中美AI竞赛：技术铁幕正在形成？
- AI 泡沫还是革命？2万亿美元赌注的真相
- 全球关税战下的科技产业链重构
- AI与就业：谁会失业，谁会崛起
- AI治理：欧盟、中国、美国三条路线
- AI 时代的教育该怎么做——个人和国家层面
- 后全球化时代，AI加速了分裂还是连接？

## 三、节目形式（三种模板）

### 形式 1：独白/深度解读（Solo Episode）

> 你一个人讲，适合支柱 A 和支柱 C

| 要素 | 内容 |
|------|------|
| 时长 | 20-35 分钟 |
| 频率 | 每月 2-3 期 |
| 结构 | 见下方模板 |
| 特点 | 强观点输出，建立个人IP |

**单期结构模板：**

```
[开场] 1-2 分钟
  - Hook：一个引发好奇的问题/事件/数据
  - "大家好，欢迎来到 AI+，我是 XX。今天聊……"

[背景铺垫] 3-5 分钟
  - 这件事/趋势的来龙去脉
  - 关键事实和数据

[核心观点] 10-15 分钟
  - 2-3 个层次递进的观点
  - 每个观点配一个案例或类比
  - 适度表达个人判断，但保留开放性

[延伸思考] 5-8 分钟
  - 这对普通人意味着什么？
  - 反直觉的角度 / 容易被忽略的维度
  - 连接到更大的图景

[收尾] 1-2 分钟
  - 总结核心takeaway（不超过3点）
  - 留一个问题给听众思考
  - 引导互动/预告下期
```

### 形式 2：跨界对谈（Guest Episode）

> 邀请行业嘉宾，适合支柱 B

| 要素 | 内容 |
|------|------|
| 时长 | 40-60 分钟 |
| 频率 | 每月 1-2 期 |
| 结构 | 见下方模板 |
| 特点 | 碰撞感、故事性、真实体验 |

**单期结构模板：**

```
[暖场] 3-5 分钟
  - 嘉宾自我介绍（不是念简历，而是"你怎么走到今天这一步的"）
  - 一个轻松的破冰问题

[行业现状] 8-12 分钟
  - 你的行业现在最大的痛点/变化是什么？
  - AI进入之前，这个行业是怎样运转的？

[AI 碰撞] 15-20 分钟（核心段）
  - AI 在你的工作中具体怎么用的？
  - 最让你惊喜/震惊的一次AI体验
  - AI 目前做不到什么？边界在哪？
  - 你的同行对AI是什么态度？（拥抱/恐惧/观望）

[未来展望] 8-10 分钟
  - 3年后你的行业会变成什么样？
  - 哪些岗位/技能会变得更重要？
  - 给想进入这个行业的人什么建议？

[快问快答] 3-5 分钟
  - 你最常用的AI工具是？
  - 一句话，AI对你的行业意味着什么？
  - 推荐一个大家应该关注的趋势

[收尾] 1-2 分钟
  - 感谢嘉宾，预告下期
```

### 形式 3：热点速评 / 月度回顾（Express Episode）

> 短小精悍，追热点或做月度总结

| 要素 | 内容 |
|------|------|
| 时长 | 10-15 分钟 |
| 频率 | 按需（大事件时出/每月底出） |
| 结构 | 3-5个要点快速过 |
| 特点 | 时效性强、信息密度高 |

**单期结构模板：**

```
[开场] 1 分钟
  - "AI+ 月度快报/热点速评，X 分钟带你过完本月最重要的事"

[要点逐条] 每条 2-3 分钟
  - 发生了什么（事实）
  - 为什么重要（分析）
  - 我的一句话判断

[总结] 1 分钟
  - 本月关键词 / 一个趋势判断
```

## 五、嘉宾策略

### 嘉宾画像（不同圈层）

```
第一梯队（容易邀请，快速启动）：
  ├── 身边的朋友/同事中在各行业的从业者
  ├── 技术社区活跃的开发者、产品经理
  └── AI工具的重度用户（各行业）

第二梯队（节目有一定影响力后）：
  ├── 行业KOL、自媒体创作者
  ├── 创业公司创始人（AI赛道或被AI影响的赛道）
  └── 投资人（关注AI方向的）

第三梯队（品牌力足够时）：
  ├── 大厂AI负责人
  ├── 学术界研究者
  └── 政策制定相关人士
```

### 嘉宾邀约话术要点

- 明确告知节目定位和听众群体
- 强调"不是技术科普，是真实体验分享"
- 提前给嘉宾一个简单的问题大纲（降低参与门槛）
- 承诺成片前给嘉宾确认（建立信任）

### 嘉宾储备清单（按行业建立）

建议维护一个表格，字段包括：

| 姓名 | 行业 | 职位 | AI使用程度 | 联系方式 | 状态 | 备注 |
|------|------|------|-----------|---------|------|------|
| ... | 医疗 | 主治医师 | 日常使用AI辅助诊断 | ... | 待邀约 | ... |

## 六、差异化与个人IP

### 你的独特价值

播客太多了，听众为什么听你的？需要想清楚这几个问题：

1. **你的视角独特性**：你是从什么角色/背景出发看AI的？（技术人、创业者、跨界者？）
2. **你的信息差**：你能接触到什么别人接触不到的人、信息、行业？
3. **你的表达风格**：严肃深度 vs 轻松幽默 vs 冷静理性？

### 建议打造的标签

选择 1-2 个持续强化：
- "跨界视角" — 不只是技术人聊技术，而是把AI放在更大的世界里看
- "真实体验" — 不讲概念，讲真实使用AI的故事和变化
- "有观点" — 不做中立搬运工，敢于给出判断（但保持谦逊）

## 八、启动清单 Checklist

### 上线前准备

- [ ] 确定节目名称和视觉VI（封面、配色）
- [ ] 录制并剪辑第0期（trailer/先导片，2-3分钟，介绍节目是什么、为什么做、听众能得到什么）
- [ ] 准备好前3期的选题和大纲
- [ ] 注册各平台账号（小宇宙、喜马拉雅、Apple Podcasts等）
- [ ] 锁定第一位嘉宾（建议选熟悉的朋友，降低首次对谈压力）
- [ ] 购买/准备基础录音设备

### 首月目标

- [ ] 发布4期节目（2 Solo + 1 嘉宾对谈 + 1 月度回顾）
- [ ] 建立稳定的制作流程（录制→剪辑→发布，单期控制在4小时内）
- [ ] 获取早期听众反馈，迭代优化

### 首季（Q2）目标

- [ ] 完成 10-12 期节目
- [ ] 覆盖至少 4 个不同行业的嘉宾
- [ ] 形成稳定的节目风格和个人IP
- [ ] 建立听众社群（微信群/即刻等）
