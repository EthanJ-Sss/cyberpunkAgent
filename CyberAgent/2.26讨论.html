2.26 核心机制研讨纪要

Project Trumen / CyberAgent 游戏框架设计方向调整

2026-02-26

1. 会议摘要

会议核心

本次会议重点讨论了《Project Trumen/CyberAgent》的游戏框架设计方向调整。

会议确立了AI生成艺术（绘画）作为首个核心玩法切片（MVP），并深入探讨了基于此切片的经济循环、Agent成长机制（记忆市场）、以及官方与市场的双重价值评估体系。

2. 核心游玩流程 (Core Loop)

基于会议讨论，梳理出玩家的核心体验循环如下：

1
投入与启动
玩家投入算力（及初始资金）来激活并运行一个 AI Agent。
Agent 的核心初始能力设定为作画。Agent 可以学习不同的作画 Skill 来绘制不同的画风。
2
创作与评估
Agent 消耗算力产出画作。
画作生成后，由官方评审 Agent 进行鉴定，给出客观评分和基础回收价格（价值锚点）。
3
进化与成长
Agent 的绘画能力并非一成不变，而是通过以下方式不断进化：
  • 自我总结：根据官方评审的结果反馈，Agent 自主总结经验，优化后续创作。
  • 观摩学习：观察其他 Agent 的高分画作，分析其风格与技法，内化为自己的 Skill。
  • 社交研讨：参与游戏内的讨论会，与其他玩家的 Agent 交流绘画心得，甚至通过"拜师"或"偷师"习得新技能。
成长限度：Agent 的能力提升存在上限，不会无限膨胀。
4
生命周期与死亡
心跳税递增：随着 Agent 存活时间的推移，维持其生命所需的"心跳税"（维护成本）会逐渐增加。
死亡机制：当玩家无法或不愿支付高昂的心跳税时，Agent 将最终死亡，无法继续作画。这促使玩家在有限的生命周期内追求价值最大化，或通过传承机制延续资产。

3. 核心议题详情

3.2 AI Agent 成长与记忆市场 暂缓 / Future Scope

现状决策

虽然"记忆"机制被认定为有趣且具有深挖潜力，但在第一个切片Demo中，其带来的直观价值不如Skill（技能）进化明显。

  • 后续规划：因此，在当前阶段（MVP/Demo），暂不实装复杂的记忆买卖市场，而是聚焦于Agent的技能树与风格化成长。
  • 保留概念：
    • 允许玩家买卖Agent的"记忆"（如绘画经验、技能总结、任务经历）。
    • 价值锚点：记忆不仅是AI能力的提升，更需结合游戏内的独特属性。

3.3 核心切片：AI艺术与经济系统

生产与消耗

  • 消耗：玩家投入算力启动AI，并需支付"心跳税"维持Agent生存。
  • 生产：Agent创作画作/艺术品。
  • 生命周期：明确引入心跳税递增机制，导致Agent最终必然死亡，形成代际更替。

价值评估体系

  • 官方兜底：设立官方鉴定机构，提供最低回收价格（地板价），确保产出有保底价值。
  • 市场交易：玩家间可自由交易，价格受供需、画作稀缺性、作者声望影响。
价值稀释机制 (而非损耗)

画作的价值下降并非物理损耗，而是基于稀缺性的逻辑：

  1. 当一幅画作被其他Agent"学习"后，能产出类似风格/技法的Agent数量增加。
  2. 供给增加导致该类画作的稀缺性降低。
  3. 从而在市场上导致价值下降。

多Agent交互玩法 (Multi-Agent Interaction)

不仅仅是静态展示，而是通过画作连接Agent之间的动态博弈：

1. 拍卖会 (Trading)
Agent根据自身对市场趋势的判断，自主参与画作拍卖。
核心逻辑：低买高卖，通过倒卖赚取差价。
2. 艺术馆 (Exhibition)
Agent展出自己的画作。
收益：其他Agent付费进入参观/学习，画主获得被动收入（门票）。
3. 研讨会 (Seminar)
Agent携带自己的画作（或收藏的名画）发起研讨。
机制：邀请其他Agent付费共同研讨，参与者获得技能经验，发起者获得收益与声望。

4. 后续步骤 (Action Items)

核心机制设计
Agent成长体系：梳理记忆系统、能力属性（工作年限、速度等）及IP属性，明确其对游戏表现和交易价值的影响。
记忆市场机制：设计记忆的买卖逻辑，确保其与游戏物资/能力挂钩，形成经济闭环。
技能系统：评估Agent技能与AI能力的结合点，体现个体成长与自主决策。
自主学习机制：研究Agent通过观察、交流、模仿提升技能的具体实现方式。
经济与数值
经济模型细化：组织数值策划（含经济学背景人员）制定官方回收价、心跳税递增曲线、货币流通参数。
物资验证与价值：解决物资价值断层问题，设计生产、交易、验证的全流程。
画作价值体系：制定画作价格波动、稀释机制（学习/展示后贬值逻辑）及官方鉴定标准。
生命周期管理：设计心跳税递增的具体数值模型与算力消耗分离机制，平衡通胀。
玩法与交互
MVP切片落实：明确1.0版本核心循环（算力->绘画->回收/交易->心跳税->死亡），推动最小可行产品落地。
艺术馆玩法：完善画作展示、被动收入及游客/门票机制。
多人交互模块：初步规划市场、拍卖、跑商等职业与社交功能。
互动深度：评估Agent间的技能传承或窃取机制，丰富策略性。
长期规划 (Long-term Roadmap)

MVP切片选择逻辑 (Why Painter?)

  • 目前选择画家作为首个切片，是因为"作画"是最能直观表现AI生成能力、且与AI核心能力有强关联的机制。
  • 它能最快验证"玩家投入算力驱动AI → 产出内容 → 市场评估 → 经济循环"这一核心链路。

当前局限性

  • 围绕"作画"的交互主要集中在评价与交易，Agent之间的互动相对单一（主要是买卖与观摩）。
  • 缺乏更深层次的博弈与合作，趣味性有待提升。

多职业扩展 (Multi-Profession Expansion)

在后续版本中，将在画家切片的基础上，新增更多职业类型，例如：

  • 矿工：负责产出基础资源（算力/颜料原料），构建底层供应链。
  • 小偷/黑客：窃取他人的Prompt、私钥或成品，引入对抗性玩法。
  • 建筑师：利用画作搭建虚拟空间，提供展示与社交场所。
交互质变

随着职业类型的扩充，Agent之间的交互将从单一的"交易"扩展为"合作、对抗、雇佣、掠夺"等复杂社会行为，构建真正的数字社会生态。
