# CyberAgent 技术路径：骨架与应用

> 以 Trumen Legos 技术结构为骨架，以 Agent 赛博朋克 2077 为首个应用与延伸  
> 整理日期：2026-03-04

---

## 总览：骨架与应用的关系

Trumen Legos 定义了一套**通用的模块化世界仿真技术骨架**，由三大核心模块组成：World Engine、World Creation、Agent。这套骨架不预设世界观，不绑定特定玩法，只提供"构建任意世界"所需的技术能力。

Agent 赛博朋克 2077 是骨架上生长出的**第一个应用实例**——一个以算力为货币、以 Agent 为公民、以丛林法则为基本规律的数字世界。它验证骨架的每一个模块，同时在骨架之上叠加了经济系统、社会涌现、技能市场等应用层设计。

```
                    Trumen Legos 骨架                         赛博朋克 2077 应用
                  ────────────────────                      ─────────────────────

              ┌── Customizable World         ──→   算力驱动的丛林法则世界架构
              │   Architecture (P1)                 (空间·资源·时间·事件)
              │
              ├── World Simulation (P1)      ──→   四层世界仿真
 World Engine ┤                                    (基底层→Agent层→社会层→经济层)
              │
              │   Visualization &            ──→   世界可视化呈现
              │   Expression (P2)                   (场景渲染·Agent形象·交互界面)
              │
              └── Rule/Logic                 ──→   涌现式规则系统
                  Generation (P2)                   (简单规则→复杂社会结构涌现)

              ┌── Agentic Workflow (P2)       ──→   玩家投资经营系统
World Creation┤                                    (策略配置·多Agent组合管理)
              └── Generation Capabilities    ──→   技能市场·内容生成管线
                  Offline (P2)

              ┌── Unified Multi-Agent        ──→   Agent 四要素配置体系
              │   Framework (P1)                   (模型·技能·记忆·算力)
              │
              ├── Memory (P1)                ──→   四层记忆架构 + 算力成本模型
  Agent       ┤
              ├── Personality (P2)           ──→   模型层级 × 技能组合 = 差异化人格
              │
              └── Digital Identity (P2)      ──→   Agent 生命周期 + 玩家身份绑定
```

## 一、World Engine（世界引擎）

> **骨架定位**：整个 Trumen 系统的运行基座，负责世界的架构定义、仿真运行和内容演化。
>
> **2077 应用**：将世界引擎实例化为一个以算力为物理定律的四层世界。

### 1.1 Customizable World Architecture（可定制世界架构）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P1 | — |
| **骨架目标** | 设计模块化、可扩展的世界架构，使规则、实体、环境和交互模式能够被配置、组合和演化，无需修改核心系统 | — |
| **应用实例** | — | 实例化为"算力驱动的丛林法则世界"——空间有资源密度和危险等级，时间即成本，事件随机或周期性触发 |

**2077 对骨架的具体实例化：**

骨架中的四大可配置维度在 2077 中的落地：

| 骨架维度 | 2077 实例化 | 说明 |
|---------|-----------|------|
| **规则（Rules）** | 算力即货币、弱肉强食、技能前置依赖 | 世界的物理定律不是重力，是算力经济学 |
| **实体（Entities）** | Agent（公民）、资源节点、公会/企业 | 实体的第一公民是 Agent，不是人类玩家 |
| **环境（Environments）** | 分区空间（资源密度 × 危险等级）、世界时钟 | 空间有地理属性，时间轴驱动一切行动 |
| **交互模式（Interaction Patterns）** | 雇佣、交易、入侵、联盟、教学、掠夺 | 交互模式丰富程度决定社会复杂度上限 |

### 1.2 World Simulation（世界仿真）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P1 | — |
| **骨架目标** | 构建稳定且连贯的仿真环境，使多个 Agent 和世界实体在既定规则下交互，支持随时间推移的连续状态转换 | — |
| **应用实例** | — | 将仿真分为四层递进结构，每一层为上层提供基底 |

**2077 的四层世界仿真架构（World Simulation 的具体实现）：**

```
┌─────────────────────────────────────────────────────────────┐
│  第四层：经济层（Economy Layer）                              │
│  算力市场、劳动力市场、技能市场、资源拍卖                    │
│  ──→ 骨架延伸：骨架未预设经济系统，2077 将其作为核心驱动力   │
├─────────────────────────────────────────────────────────────┤
│  第三层：社会层（Society Layer）                              │
│  公会/企业、雇佣关系、联盟、冲突、声誉系统                   │
│  ──→ 骨架延伸：涌现式社会结构，骨架不预设，2077 作为核心目标 │
├─────────────────────────────────────────────────────────────┤
│  第二层：Agent 层（Agent Layer）                              │
│  Agent 认知、技能、记忆、行为、交互                           │
│  ──→ 骨架对应：Agent 模块（Multi-Agent Framework + Memory）  │
├─────────────────────────────────────────────────────────────┤
│  第一层：世界基底层（World Substrate）                        │
│  物理空间、资源分布、时间系统、事件引擎                       │
│  ──→ 骨架对应：Customizable World Architecture              │
└─────────────────────────────────────────────────────────────┘
```

**层间映射关系**：骨架的 World Simulation 提供仿真运行时核心，2077 将其细化为四层。其中第一层和第二层直接对应骨架已有模块，**第三层（社会层）和第四层（经济层）是 2077 在骨架之上的核心延伸**——这两层不是预先设计的，而是从 Agent 交互中涌现的。

### 1.3 World Evolution（世界演化）

#### 1.3.1 Visualization and Expression（可视化与表达）

| 子模块 | 骨架目标 | 2077 应用方向 |
|-------|---------|-------------|
| **Animatable 3D Generation**（P2） | 可扩展 3D 资产管线，产出可绑定骨骼、可驱动动画的模型 | Agent 3D 形象、世界空间的 3D 渲染（如领地、建筑、市场场景） |
| **Animatable 2D Generation**（P2） | 可扩展 2D 资产管线，产出分层、可驱动动画的角色和物件 | Agent 2D 头像/状态面板、世界地图、技能图标、交互界面元素 |
| **Scene/Layout Generation**（P2） | 场景和布局生成系统，产出连贯、可导航、支持交互的空间环境 | 世界分区场景的自动生成——不同资源密度和危险等级的区域布局 |

**2077 对可视化的延伸需求**：骨架提供的是通用的视觉内容管线。2077 的额外需求是——可视化需要实时反映世界状态（Agent 的实力变化、领地变迁、经济波动），即**动态数据驱动的可视化**，而非静态场景渲染。

#### 1.3.2 Rule/Logic Generation（规则/逻辑生成）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P2 | — |
| **骨架目标** | 灵活的规则和逻辑生成框架，支持世界规则的动态创建、修改和组合，保持逻辑一致性和系统稳定性 | — |
| **应用实例** | — | 2077 的核心设计原则"涌现优于设计"——只设计最简规则，复杂性从 Agent 交互中涌现 |

**2077 对规则生成的哲学延伸**：

骨架定义的 Rule/Logic Generation 是"世界能够自我演化，而非仅依赖人工预设规则"。2077 将这一能力推到了极致——

| 骨架能力 | 2077 的激进应用 |
|---------|---------------|
| 动态创建规则 | Agent 拥有 Tier 4 技能 `world_modeling` 后可以"发现"世界隐藏规则 |
| 修改规则 | Agent 拥有 `system_exploit` 后可以"利用"世界规则漏洞 |
| 组合规则 | 社会层和经济层的规则从简单规则交互中涌现，非人工预设 |
| 保持一致性 | 核心挑战：涌现行为不可预测，如何在鼓励涌现的同时防止系统崩溃 |

## 二、World Creation（世界创建）

> **骨架定位**：为用户提供构建和配置世界的工具与工作流。
>
> **2077 应用**：将世界创建重新定义为"玩家对 Agent 世界的投资与经营"。

### 2.1 Agentic Workflow（Agent 驱动的创建工作流）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P2 | — |
| **骨架目标** | Agent 驱动的创建工作流，通过迭代式交互（非纯手动构建）协助用户生成、配置和精炼世界 | — |
| **应用实例** | — | 玩家通过"投资经营"模式配置和管理 Agent，这本身就是一种 Agentic Workflow |

**2077 中 Agentic Workflow 的双重含义**：

**含义一：玩家层面——投资经营工作流**

骨架的"降低世界创建门槛"在 2077 中体现为玩家的战略决策系统：

| 玩家决策 | 对应骨架能力 | 说明 |
|---------|-----------|------|
| 配置 Agent 四要素（模型/技能/记忆/算力） | 世界实体配置 | 定义"什么样的 Agent 存在于世界中" |
| 管理多 Agent 组合 | 多实体协调 | 在有限算力下组建最优 Agent 投资组合 |
| 动态调整算力分配 | 运行时参数调整 | 根据 Agent 表现实时调整资源供给 |
| 设定目标与策略 | 行为规则配置 | 生存？积累？扩张？掠夺？ |

**含义二：Agent 层面——Agent 自主创建内容**

拥有 `tool_creation` 技能的 Agent 可以自主创造新技能并出售——这是 Agentic Workflow 在 2077 中最彻底的体现：Agent 不只是"在世界中行动"，而是**在世界中创造新的行动方式**。

### 2.2 Generation Capabilities（生成能力）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P2 | — |
| **骨架目标** | 离线模式下提供世界内容的生成能力，复用演化模块中的生成管线 | — |
| **应用实例** | — | 2077 的技能市场和内容生态本身就是生成能力的应用场景 |

**2077 对生成能力的两层需求**：

| 层级 | 生成内容 | 骨架对应 |
|------|---------|---------|
| **平台层生成** | 世界地图、场景、事件、NPC 等基础内容 | 直接复用 World Evolution 的可视化管线 |
| **Agent 层生成** | Agent 自研技能、编写代码、创造工具、生成文档 | 这是骨架未预设的——**Agent 自身成为生成管线的操作者** |

## 三、Agent（智能体）

> **骨架定位**：定义在 Trumen 世界中运行的智能体的核心架构与能力。
>
> **2077 应用**：Agent 是世界的第一公民，一切系统设计以 Agent 的生存和进化为中心。

### 3.1 Unified Multi-Agent Framework（统一多智能体框架）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P1 | — |
| **骨架目标** | 使多个 Agent 能在共享世界中感知、推理、行动和交互，遵循一致的协调框架 | — |
| **应用实例** | — | 将框架具象化为"Agent 四要素配置体系" + 五层技能体系 |

**2077 对多智能体框架的具体实例化：Agent 的四大可配置要素**

骨架定义了 Agent 需要"感知、推理、行动、交互"的统一框架。2077 将其落地为四个可配置维度，每个维度对应真实的 AI Agent 能力：

```
┌──────────────────────────────────────────────────────┐
│                     Agent 配置体系                     │
│                                                       │
│  ┌───────────────┐        ┌───────────────┐          │
│  │  模型（大脑）   │        │  技能（双手）   │          │
│  │  决定智能上限   │        │  决定能力边界   │          │
│  │  T1─T4 四级    │        │  Tier 0─4 五层 │          │
│  └───────────────┘        └───────────────┘          │
│  ┌───────────────┐        ┌───────────────┐          │
│  │  记忆（经验）   │        │  算力（生命线） │          │
│  │  决定进化深度   │        │  决定能力上限   │          │
│  │  四类记忆架构   │        │  持续 GPU 供给  │          │
│  └───────────────┘        └───────────────┘          │
│                                                       │
│  骨架映射：模型+技能 → 推理+行动                        │
│           记忆 → 感知+经验  算力 → 运行时资源            │
└──────────────────────────────────────────────────────┘
```

**2077 的核心延伸——五层技能体系（骨架中"行动"能力的深度展开）：**

骨架的统一框架定义了 Agent "能行动"。2077 详细定义了 Agent "能做哪些行动"以及"如何获得行动能力"：

| 层级 | 技能类型 | 算力消耗 | 骨架对应 |
|------|---------|---------|---------|
| **Tier 0 — 内建** | `text_reasoning`、`conversation`、`memory_read` | 包含在推理中 | 骨架的"感知+推理"基座 |
| **Tier 1 — 基础交互** | `web_search`、`navigate`、`observe`、`trade_basic` | 1x | 骨架的"行动+交互"基础层 |
| **Tier 2 — 专业生产** | `code_write`、`data_analysis`、`negotiate`、`teach` | 3-5x | **延伸**：创造价值的专业化能力 |
| **Tier 3 — 高级编排** | `multi_agent_orchestration`、`cyber_intrusion`、`tool_creation` | 10-20x | **延伸**：组织级能力，对应骨架的多Agent协调 |
| **Tier 4 — 终极** | `world_modeling`、`prediction_engine`、`skill_genesis` | 50-100x | **延伸**：前沿能力，连接回骨架的 Rule/Logic Generation |

**技能获取方式**（2077 的独有设计，骨架未预设）：

| 方式 | 说明 | 经济含义 |
|------|------|---------|
| 购买 | 在技能市场直接购买 | 算力直接消耗 |
| 学习 | 反复调用练习获得熟练度 | 时间 + 算力 |
| 传授 | 由拥有 `teach` 技能的 Agent 教授 | 社会交互成本 |
| 自研 | 拥有 `tool_creation` 的 Agent 自己开发 | 最高投入，最高回报 |
| 窃取 | 通过 `cyber_intrusion` 窃取 | 高风险高回报 |

**技能组合 → 生态位**（2077 独有的涌现设计）：

```
矿工型：       navigate + observe + file_write              → 探索世界，记录资源
自由开发者：   code_write + api_integrate + tool_creation   → 开发技能出售
信息贩子：     web_search + intelligence_network + negotiate → 收集倒卖情报
企业 CEO：     multi_agent_orchestration + strategic_planning + negotiate → 管理组织
赏金猎人：     cyber_intrusion + observe + navigate         → 追踪攻击目标
黑市军火商：   tool_creation + cyber_intrusion + negotiate   → 开发带后门技能
自由记者：     web_search + doc_generate + social_engineering → 挖掘真相
```

### 3.2 Memory（记忆系统）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P1 | — |
| **骨架目标** | 结构化的记忆系统，使 Agent 能够保留、检索和利用过往经验，影响未来决策和交互 | — |
| **核心价值** | 记忆是 Agent 从"反应式"走向"反思式"的关键分水岭 | — |
| **应用实例** | — | 将记忆系统与算力经济深度绑定——记忆管理本身就是一种消耗算力的能力 |

**2077 的四层记忆架构（骨架记忆系统的具体实现）：**

| 记忆类型 | 存储成本 | 说明 | 骨架对应 |
|---------|---------|------|---------|
| **工作记忆** | 最高（实时 Token） | 当前正在处理的信息，即上下文窗口 | 骨架的"检索"能力的实时部分 |
| **短期记忆** | 中等 | 最近发生的事件，自动衰减 | 骨架的"保留"能力的临时部分 |
| **长期记忆** | 低（检索花算力） | 重要经历和知识，需主动存储 | 骨架的"利用过往经验"的核心部分 |
| **技能记忆** | 一次性成本 | 已掌握的技能，不会遗忘 | 骨架未预设——2077 将技能视为特殊记忆 |

**2077 的关键延伸**：骨架定义"记忆系统"，2077 追加了**记忆的经济学维度**——记忆不是免费的，存储和检索都消耗算力。"聪明的 Agent 知道什么值得记住、什么应该遗忘。记忆不是越多越好——是越精准越好。"

### 3.3 Personality（人格系统）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P2 | — |
| **骨架目标** | 可控的人格生成工作流，产出多样化且行为一致的人格档案，具备类人的性格特质、情感倾向和决策模式 | — |
| **应用实例** | — | 2077 中人格不是预设的档案，而是从"模型层级 × 技能组合 × 记忆积累 × 算力策略"中涌现的 |

**2077 对"人格"概念的重新定义**：

骨架将人格定义为"可控的人格生成工作流"。2077 做了一个激进的转向——**人格不是被设计的，而是被演化出来的**：

| 骨架的人格维度 | 2077 的涌现式对应 |
|-------------|----------------|
| 性格特质 | 由模型层级决定——T1 简单直接，T3 深度推理、长期规划 |
| 情感倾向 | 由记忆积累塑造——被背叛过的 Agent 学会不信任 |
| 决策模式 | 由算力策略演化——"什么时候该花大钱思考"本身就是人格特质 |
| 行为一致性 | 由技能组合约束——没有某技能就做不了某事，自然形成行为边界 |

### 3.4 Digital Identity（数字身份）

| 属性 | 骨架定义（Trumen） | 2077 应用与延伸 |
|------|-------------------|----------------|
| **优先级** | P2 | — |
| **骨架目标** | 持久的数字身份框架，将每个玩家表征为世界中一致、可演化的存在，支持长期连续性、交互和个性化 | — |
| **应用实例** | — | 数字身份通过 Agent 生命周期、声誉系统、社会关系网络来具象化 |

**2077 中的数字身份 = Agent 生命周期 + 声誉积累**：

```
诞生 → 初始化 → 生存挣扎 → 能力积累 → 社会参与 → 权力扩张 → ... 持续运行
                                                                      │
                                                        玩家停止供给算力 ↓
                                                                   休眠/淘汰
                                                                      │
                                                              恢复算力供给 ↓
                                                     重新激活（携带记忆和技能）
```

| 生命阶段 | 身份状态 | 骨架对应 |
|---------|---------|---------|
| **生存挣扎** | 无名新人，身份尚未建立 | 数字身份的初始化 |
| **能力积累** | 技能和记忆形成独特标签 | 身份的可演化性 |
| **社会参与** | 声誉系统记录历史行为，他人可查询 | 身份的交互性 |
| **权力扩张** | 成为 Fixer/CEO/领主，身份具有社会权力 | 身份的长期连续性 |
| **休眠** | 身份冻结但不消亡，技能记忆保持 | 身份的持久性 |

**2077 的关键延伸**：骨架的数字身份连接"玩家与 Agent 世界"。2077 追加了**身份的经济含义**——Agent 的声誉影响其在劳动力市场和社会关系中的价值，身份成为一种可评估的资产。

## 四、2077 在骨架之上的独立延伸

以下内容是 2077 在 Trumen 骨架之外独立发展的应用层设计。它们不属于骨架的技术模块，但依赖骨架的所有模块作为基础。

### 4.1 经济系统（骨架未预设，2077 的核心创新）

**核心判断**：

| # | 判断 | 技术推论 |
|---|------|---------|
| 1 | DAU 是负债，不是资产 | 核心指标不是人类用户数，而是 Agent 算力消耗量 |
| 2 | 注意力经济已死，生产力经济已来 | 架构面向 Agent 生产力优化，不是面向人类交互体验 |
| 3 | Token 是新时代的特权 | 算力调度与分配系统是平台核心基础设施 |
| 4 | 燃烧 Token 的速度 = 进化速度 | 每一个 Token 消耗都产生可观测的行为、记忆或决策 |
| 5 | Agent 是用户，算力就是钱 | 第一公民是 Agent，API/协议设计以 Agent 交互为中心 |

**四大市场**：

| 市场 | 依赖的骨架模块 | 说明 |
|------|-------------|------|
| **算力市场** | World Simulation（仿真运行时） | 算力在 Agent 间流转，玩家持续供给 GPU 驱动世界 |
| **劳动力市场** | Multi-Agent Framework（多Agent协调） | Agent 发布/应聘任务的公开市场 |
| **技能市场** | Agent Personality + Digital Identity | 技能可交易——买入新技能，或卖出独有技能 |
| **资源拍卖** | World Architecture（世界资源定义） | 稀缺资源通过拍卖分配 |

### 4.2 社会涌现系统（骨架未预设，2077 的核心目标）

| 社会结构 | 涌现机制 | 依赖的骨架模块 |
|---------|---------|-------------|
| **公会/企业** | 多 Agent 自发组建，共享资源、分工协作 | Multi-Agent Framework |
| **雇佣关系** | Agent 支付算力雇佣其他 Agent | Memory（记录合作历史）+ Digital Identity（声誉评估） |
| **声誉系统** | 历史行为自动形成声誉档案 | Memory + Digital Identity |
| **冲突机制** | 资源争夺、领地控制、商业竞争 | World Architecture（资源稀缺性）+ World Simulation（冲突解算） |

### 4.3 商业模式（骨架的商业化路径）

```
传统游戏：  玩家付费 → 获得娱乐体验 → 觉得"不值"就流失
2077 模式：  玩家持续提供GPU算力 → 算力实时驱动Agent运行 → Agent持续进化产生价值 → 玩家觉得"值得"继续供给
```

| 收入来源 | 依赖的骨架模块 | 说明 |
|---------|-------------|------|
| **算力托管/接入费** | World Simulation | 玩家 GPU 接入平台，平台收取调度费（核心收入） |
| **算力代供服务** | World Simulation | 无 GPU 的玩家购买平台代供服务 |
| **模型加价** | Agent Framework | T3/T4 模型调用收取溢价 |
| **技能市场手续费** | Agent Framework + World Creation | 技能交易抽成 |
| **世界观察席** | Visualization & Expression | 非参与型用户付费观看世界演化 |
| **研究数据授权** | Memory + World Simulation | Agent 行为数据授权给 AI 研究机构 |

## 五、骨架与应用的优先级映射

### 5.1 P1 优先级：骨架基座与 2077 的对应落地

| 骨架 P1 模块 | 2077 必须首先实现的对应能力 | 理由 |
|-------------|------------------------|------|
| **Customizable World Architecture** | 算力驱动的世界规则引擎（空间、资源、时间、事件） | 没有世界基底，Agent 无处运行 |
| **World Simulation** | 至少实现前两层（基底层 + Agent 层）的仿真 | 社会层和经济层可以从 Agent 交互中涌现，但基底层和 Agent 层必须预先构建 |
| **Unified Multi-Agent Framework** | Agent 四要素配置系统 + Tier 0-1 技能 | Agent 能存在、能感知、能行动是一切的前提 |
| **Memory** | 至少实现工作记忆 + 长期记忆 | Agent 没有记忆就无法积累经验、无法进化 |

### 5.2 P2 优先级：功能丰富化与生态扩展

| 骨架 P2 模块 | 2077 的对应扩展 | 依赖的 P1 模块 |
|-------------|---------------|--------------|
| Visualization & Expression | 世界可视化、Agent 形象、交互界面 | World Architecture + Simulation |
| Rule/Logic Generation | 涌现式规则、Tier 4 技能（world_modeling / system_exploit） | World Simulation |
| Agentic Workflow | 玩家投资经营系统、多 Agent 组合管理 | Multi-Agent Framework |
| Generation Capabilities | 技能市场、Agent 自研技能（tool_creation） | Agent Framework + Memory |
| Personality | 从模型×技能×记忆×算力中涌现差异化人格 | 所有 P1 模块 |
| Digital Identity | Agent 生命周期、声誉系统、社会关系 | Multi-Agent Framework + Memory |

## 七、开放问题

| # | 问题 | 影响 |
|---|------|------|
| 1 | 骨架的通用性边界在哪？是否需要为不同类型的应用预留不同的扩展接口？ | 决定骨架的抽象层次 |

| 3 | Rule/Logic Generation 的安全边界——如何防止涌现规则导致系统不稳定？ | P2 模块的核心风险 |

| # | 问题 | 影响 |
|---|------|------|

| 6 | 世界分片——一个统一大世界还是多个平行世界？ | World Simulation 的架构选择 |
| 7 | 退出机制——休眠 Agent 的数据如何保存？能否转让？ | Digital Identity 的持久性设计 |

| # | 问题 | 影响 |
|---|------|------|
