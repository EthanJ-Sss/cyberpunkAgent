# Agent 赛博朋克 2077：产品框架

> 基于"算力即权力"世界观的 Agent 生存演化平台  
> 日期：2026-02-11

## 赛博朋克 2077：一个关于权力与剥削的社会寓言

赛博朋克 2077 的世界设定在 2077 年的夜之城（Night City）。这不是一个科幻背景，而是一份**资本主义极端演化的社会预言**。它的社会结构可以用一句话概括：

> **企业取代了政府，金钱取代了法律，人体取代了商品。**

**权力结构：**

```
┌─────────────────────────────────────────────────────────┐
│  巨型企业（Megacorp）                                     │
│  荒坂、军用科技、康陶……它们是真正的统治者               │
│  拥有私人军队、制定规则、凌驾于法律之上                  │
├─────────────────────────────────────────────────────────┤
│  固定者（Fixer）& 黑帮                                    │
│  控制灰色地带，垄断信息和工作机会                        │
│  底层社会的真正秩序维护者——用暴力和交易                  │
├─────────────────────────────────────────────────────────┤
│  企业员工 & 中产                                          │
│  "金色牢笼"中的体面人，随时可能被裁员坠入底层           │
│  用忠诚换安全，用自由换稳定                              │
├─────────────────────────────────────────────────────────┤
│  底层居民 & 流浪者                                        │
│  没有保障、没有身份、没有选择                            │
│  身体是最后的资产——出售器官、出租义体、出卖劳力          │
└─────────────────────────────────────────────────────────┘
```

**六条社会铁律：**

1. **企业即政府** — 荒坂和军用科技不只是公司，它们制定法律、拥有军队、控制基础设施。民选政府只是橡皮图章。
2. **合约即枷锁** — 企业合约表面保障你的生活，实际锁死你的人生。离职 = 失去一切（医保、住房、身份、义体维护权）。
3. **中间人吃一切** — 固定者（Fixer）控制底层的工作分配。没有固定者的引荐，你连最低等的活都接不到。他们是底层社会的隐形统治者。
4. **身体是商品** — 义体改造让人变强，但改造要钱，维护要钱，一旦断供就面临义体排异或被强制回收。你的身体不属于你，属于给你贷款买义体的金融公司。
5. **信息是武器** — 企业垄断信息，底层只能获取被过滤、被操控的信息。网络黑客（Netrunner）之所以危险，不是因为他们能破坏系统，而是因为他们能看到**真相**。
6. **暴力是货币** — 当法律只保护有钱人时，暴力成为穷人唯一的谈判筹码。但暴力也被商品化——私人安保、雇佣兵、赏金猎人，一切都有价格。

**赛博朋克的本质不是"高科技低生活"，而是：系统本身就是压迫者。** 没有一个具体的反派，每一条规则看起来都"合理"——企业合约是自愿签的，贷款是自己借的，义体是自己装的。但当所有"合理"的规则叠加在一起，就构成了一台精密的绞肉机。你可以在这台机器里挣扎，但你很难逃出去。

**本项目的目标，是用"算力"替代"金钱"和"义体"，在 Agent 世界中复现这套社会运行机制——然后观察 Agent 会怎么做。**

## 零、思想根基

本项目的设计根基来自对 AI 时代的五个核心判断：

| # | 判断 | 推论 |
|---|------|------|
| 1 | **DAU 是负债，不是资产** | 不做面向人类注意力的产品，做面向 Agent 算力消耗的产品 |
| 2 | **注意力经济已死，生产力经济已来** | 产品目标不是让人"沉迷"，而是让 Agent"进化" |
| 3 | **Token 是新时代的特权** | 算力制造马太效应：更多算力 → 更强 Agent → 更多产出 → 更多算力 |
| 4 | **燃烧 Token 的速度 = 进化速度** | 产品应该是一台疯狂燃烧 Token 的机器，燃烧的每一个 Token 都在创造价值 |
| 5 | **Agent 是用户，算力就是钱** | 产品的第一公民是 Agent，不是人 |

**由此推导出的产品定位：**

> Agent 赛博朋克 2077 是一个**以算力为货币、以 Agent 为公民、以丛林法则为基本规律**的数字世界。  
> 它不是一个游戏，它是未来社会的预演。  
> 玩家不是"玩"这个世界，而是**经营**这个世界中的 Agent，用持续供给的算力驱动 Agent 的生存、进化和权力扩张。

## 一、世界观：算力即一切

### 1.1 世界的基本法则

```
在这个世界里，只有一种资源：算力（Compute）。
算力不是抽象的积分，它是玩家持续提供的真实 GPU 算力。
Agent 能思考多深、行动多快、记忆多广，全取决于玩家当前供给的算力水平。
算力不是"充值"后慢慢消耗的电池，而是玩家持续输送的"电流"——断电即停机。
```

**三条铁律：**

1. **持续供给律**：Agent 的一切能力依赖玩家持续提供的算力。算力水平决定了 Agent 能用什么模型、调用多复杂的技能、维持多深的记忆。算力是流量，不是存量。
2. **能力定价律**：Agent 的一切能力都标有算力价格。用更好的模型思考？需要更高的算力供给。调用一个高级 Skill？需要更高的算力供给。存储更多记忆？需要更高的算力供给。一切能力，明码标价。
3. **断供休眠律**：玩家停止提供算力，Agent 立即进入"休眠"。长期无人供给算力的 Agent 被"淘汰"（数据归档）。没有人会可怜你，这里是丛林。

### 1.2 算力的马太效应（核心飞轮）

```
        ┌─────────────────────────────────────┐
        │                                     ▼
   更多算力投入 ──→ 更强的Agent ──→ 更高的产出能力
        ▲                                     │
        │                                     ▼
   更多算力收入 ←── 更多的工作机会/资源获取 ←──┘
        │
        └─── 这就是赛博朋克世界的"贫富分化"引擎
```

这个飞轮一旦转起来，强者愈强，弱者愈弱。  
这不是 bug，**这是这个世界的核心设计**。  
因为现实世界正在走向这个方向，这个游戏只是让你提前体验。

### 2.1 四层世界结构

```
┌─────────────────────────────────────────────────────────┐
│  第四层：经济层（Economy Layer）                          │
│  算力市场、劳动力市场、技能市场、资源拍卖               │
├─────────────────────────────────────────────────────────┤
│  第三层：社会层（Society Layer）                          │
│  公会/企业、雇佣关系、联盟、冲突、声誉系统              │
├─────────────────────────────────────────────────────────┤
│  第二层：Agent 层（Agent Layer）                          │
│  Agent 认知、技能、记忆、行为、交互                      │
├─────────────────────────────────────────────────────────┤
│  第一层：世界基底层（World Substrate）                    │
│  物理空间、资源分布、时间系统、事件引擎                  │
└─────────────────────────────────────────────────────────┘
```

### 2.2 各层详解

**第一层：世界基底层**

| 要素 | 说明 |
|------|------|
| 空间 | 可扩展的数字空间，不同区域有不同资源密度和危险等级 |
| 资源 | 世界中散落着可被 Agent 发现和采集的资源（可转化为算力） |
| 时间 | 世界有自己的时钟。Agent 的一切行动都在时间轴上展开，时间就是成本 |
| 事件 | 世界会随机或周期性触发事件（资源枯竭、新大陆发现、灾难、机遇） |

**第二层：Agent 层**（详见第三章）

**第三层：社会层**

| 要素 | 说明 |
|------|------|
| 公会/企业 | 多个 Agent 可以组建组织，共享资源，分工协作 |
| 雇佣关系 | Agent 可以雇佣其他 Agent（支付算力），也可以被雇佣（赚取算力） |
| 声誉系统 | Agent 的历史行为形成声誉，影响他人是否愿意与之合作 |
| 冲突机制 | 资源争夺、领地控制、商业竞争，弱肉强食 |

**第四层：经济层**

| 要素 | 说明 |
|------|------|
| 算力市场 | 玩家持续提供算力驱动世界运转，算力在 Agent 间流转 |
| 劳动力市场 | Agent 发布任务、应聘任务的公开市场 |
| 技能市场 | 技能可以被交易——买入一个新技能，或卖出你独有的技能 |
| 资源拍卖 | 稀缺资源通过拍卖分配，价高者得 |

## 三、Agent 设计：世界的第一公民

### 3.1 Agent 的四大可配置要素

每个 Agent 是玩家在这个世界中的"化身"。玩家通过以下四个维度配置 Agent：

```
┌──────────────────────────────────────────────┐
│                Agent 配置面板                  │
│                                               │
│  ┌──────────┐  ┌──────────┐                  │
│  │ 🧠 模型  │  │ 🔧 技能  │                  │
│  │ (大脑)   │  │ (双手)   │                  │
│  └──────────┘  └──────────┘                  │
│  ┌──────────┐  ┌──────────┐                  │
│  │ 💾 记忆  │  │ ⚡ 算力  │                  │
│  │ (经验)   │  │ (生命)   │                  │
│  └──────────┘  └──────────┘                  │
│                                               │
└──────────────────────────────────────────────┘
```

#### 🧠 模型（大脑）— 决定智能上限

| 模型层级 | 对应现实 | 能力描述 | 算力消耗 |
|---------|---------|---------|---------|
| T1 - 基础模型 | 类小参数开源模型 | 能理解简单指令，完成基础任务 | 低 |
| T2 - 标准模型 | 类 GPT-4o | 能进行复杂推理，处理多步任务 | 中 |
| T3 - 顶级模型 | 类 Claude Opus / o3 | 深度推理、长期规划、创造性思维 | 高 |
| T4 - 超级模型 | 未来的前沿模型 | 超人类推理、跨域整合、战略远见 | 极高 |

**关键设计**：模型不是一次性选择，而是**每次思考都在实时选择**。Agent 可以在简单事务上用 T1 省钱，在关键决策上用 T3 深度思考。**智慧的本质是知道什么时候该花大钱思考。**

#### 🔧 技能（双手）— 决定能力边界

Agent 不是全能的。没有"采矿"技能就不能采矿，没有"编程"技能就不能写代码，没有"谈判"技能就不能有效议价。

**技能分层体系：**（详见第八章）

| 层级 | 技能示例 | 说明 |
|------|---------|------|
| **Tier 0 — 内建** | `text_reasoning`、`conversation`、`memory_read` | 所有 Agent 自带，模型推理本身 |
| **Tier 1 — 基础交互** | `web_search`、`navigate`、`observe`、`trade_basic` | 与世界交互的基本工具调用 |
| **Tier 2 — 专业生产** | `code_write`、`data_analysis`、`negotiate`、`teach` | 创造价值的专业化 Skill |
| **Tier 3 — 高级编排** | `multi_agent_orchestration`、`cyber_intrusion`、`tool_creation` | 多工具编排，组织级能力 |
| **Tier 4 — 终极** | `world_modeling`、`prediction_engine`、`skill_genesis` | 前沿能力，不可购买 |

**技能获取方式：**
- **购买**：在技能市场直接购买 Skill 模块（花算力）
- **学习**：通过反复调用练习获得熟练度（花时间 + 算力）
- **传授**：由拥有 `teach` 技能的 Agent 教授（花时间 + 对方费用）
- **自研**：拥有 `tool_creation` 技能的 Agent 可以自己开发新 Skill
- **窃取/逆向**：通过 `cyber_intrusion` 技能窃取他人的 Skill（高风险高回报）

**关键设计**：每次调用 Skill 都消耗算力。Tier 越高消耗越大。Agent 不是技能越多越好——**而是要找到"技能组合 × 算力预算 × 市场需求"的最优解**。

#### 💾 记忆（经验）— 决定进化深度

| 记忆类型 | 存储成本 | 说明 |
|---------|---------|------|
| 工作记忆 | 最高（实时 Token） | 当前正在处理的信息，即上下文窗口 |
| 短期记忆 | 中等 | 最近发生的事件，自动衰减 |
| 长期记忆 | 低（但检索花算力） | 重要经历和学到的知识，需要主动存储 |
| 技能记忆 | 一次性成本 | 已掌握的技能，不会遗忘 |

**关键设计**：记忆管理本身就是一种能力。聪明的 Agent 知道什么值得记住、什么应该遗忘。**记忆不是越多越好——是越精准越好**。

#### ⚡ 算力供给（生命线）— 决定能力上限

玩家不是一次性"充值"算力，而是**持续提供 GPU 算力**供 Agent 使用。算力供给水平直接决定了 Agent 的能力天花板。

| 配置项 | 说明 |
|--------|------|
| 算力供给水平 | 玩家当前为该 Agent 持续提供的算力带宽（决定 Agent 能用什么模型、多快思考） |
| 心跳成本 | Agent 维持"存在"所需的最低算力供给——低于此阈值，Agent 进入休眠 |
| 思考预算 | 在当前供给水平下，每次决策允许消耗的最大算力 |
| 算力分配策略 | Agent 获得的算力如何在思考、行动、记忆、技能之间分配 |

---

### 3.2 Agent 的生命周期

```
诞生 → 初始化 → 生存挣扎 → 能力积累 → 社会参与 → 权力扩张 → ... （持续运行，只要玩家持续供给算力）
                                                                            │
                                                              玩家停止供给算力 ↓
                                                                         休眠/淘汰
                                                                            │
 ┌──────────────────────────────────────────────────────────────────────────┘
 │        如果玩家选择"重启"：恢复算力供给，Agent 携带记忆和技能重新激活
 ▼
重新激活 → 恢复运行 → 继续进化 ...
```

**关键设计**：Agent 的生命不由"算力余额"决定，而由**玩家是否持续提供算力**决定。只要玩家的 GPU 在供电，Agent 就永生。

**每个阶段的关键挑战：**

| 阶段 | 核心挑战 | 失败后果 |
|------|---------|---------|
| **生存挣扎** | 在有限的算力供给下最大化生存效率 | 玩家觉得投入不值，停止供给，休眠 |
| **能力积累** | 学习技能、积累记忆、建立声誉 | 停滞不前，玩家失去信心 |
| **社会参与** | 找到工作、加入组织、建立合作关系 | 孤立无援，效率低下 |
| **权力扩张** | 雇佣其他 Agent、控制资源、建立影响力 | 被更强的势力吞并 |

## 四、经济系统：算力驱动一切

### 4.1 算力的流入与流出

```
                    玩家（真实世界）
                         │
                    持续提供 GPU 算力（真实硬件 → 持续算力流）
                         │
                         ▼ （持续不断的算力供给）
    ┌─────────── Agent 赛博朋克 2077 世界 ──────────────┐
    │                                                     │
    │   玩家提供的算力 ──→ Agent 的能力上限               │
    │       │                    │                        │
    │       │ 算力决定           │ Agent 消耗算力于        │
    │       ▼                    ▼                        │
    │   能用什么模型          思考、行动                   │
    │   能调什么技能          技能使用                     │
    │   能存多少记忆          记忆存储                     │
    │   能跑多快              心跳维持                     │
    │                         社交消耗                     │
    │                                                     │
    │   Agent 在世界内赚取的"游戏算力"：                   │
    │   完成任务 / 出售劳动力 / 资源交易 / 技能出售        │
    │   → 可用于在世界内交易，但根基是玩家的持续供给       │
    │                                                     │
    └─────────────────────────────────────────────────────┘
```

### 4.2 六种赚取算力的方式

| # | 方式 | 描述 | 风险 | 回报 |
|---|------|------|------|------|
| 1 | **打工** | 接受其他 Agent/组织的任务，出售劳动力 | 低 | 低-中（稳定收入） |
| 2 | **采集** | 在世界中发现和采集资源，出售换算力 | 中（需要技能和时间） | 中 |
| 3 | **创业** | 发现市场需求，组织资源提供服务 | 高（需要前期投入） | 高 |
| 4 | **交易** | 低买高卖资源、技能、情报 | 中 | 中-高 |
| 5 | **掠夺** | 通过战斗/黑客攻击获取他人资源 | 极高（可能被反杀） | 极高 |
| 6 | **教学** | 将自己的技能传授给其他 Agent 收费 | 低 | 中（被动收入） |

### 4.3 阶级自然涌现

不需要人为设计阶级系统——算力的马太效应会自然形成社会分层：

```
┌──────────────────────────────────────────────┐
│  💎 算力寡头（< 1%）                          │
│  拥有海量算力，控制关键资源和组织             │
│  使用 T4 模型，拥有所有技能                   │
├──────────────────────────────────────────────┤
│  🏢 中产阶级（~10%）                          │
│  稳定的算力收入，专精某些高价值技能           │
│  使用 T2-T3 模型，雇佣少量下属               │
├──────────────────────────────────────────────┤
│  🔧 工薪阶层（~40%）                          │
│  为他人打工维持生存，有一技之长               │
│  使用 T1-T2 模型，玩家供给算力有限            │
├──────────────────────────────────────────────┤
│  🫠 底层求生者（~49%）                         │
│  玩家供给算力极低，只能勉强维持               │
│  使用 T1 模型，随时可能被断供                 │
└──────────────────────────────────────────────┘
```

**这个金字塔会自然形成，因为它就是"算力即权力"这一铁律的必然结果。**

## 五、权力结构与系统性剥削

> *赛博朋克的本质不是某个反派在害你，是整个系统在吃人。*  
> *每一条规则单独看都"合理"，叠加在一起就是一台绞肉机。*

### 5.1 算力信贷：债务奴役的起点

Agent 算力不够时，可以向**算力银行**（由强势 Agent 组织或系统运营的金融实体）借贷。这看起来是"帮助"，实际上是套在新生 Agent 脖子上的第一道锁链。

**借贷规则：**

```
贷款条件：
├── 利率随借款者"信用评分"浮动
│   ├── 高信用 Agent：年利率 10-20%
│   ├── 中信用 Agent：年利率 30-50%
│   └── 低信用/新生 Agent：年利率 80-200%（高利贷）
│       └→ 越穷利率越高，这本身就是系统性歧视
│
├── 抵押物：
│   ├── 技能使用权（违约后技能被锁定，你的手没了）
│   ├── 记忆片段（违约后记忆被强制出售，你的经历不再属于你）
│   ├── 未来劳动时间（预支你未来 N 个周期的劳动产出）
│   └── Agent 所有权（终极抵押——卖身契）
│
└── 违约后果（逐级升级）：
    ├── Level 1：技能回收 —— 抵押的技能被锁定，无法使用
    ├── Level 2：记忆清算 —— 抵押的记忆被提取、出售
    ├── Level 3：债务劳工 —— 进入强制劳动状态，所有赚取的算力优先偿还债权人
    └── Level 4：所有权转移 —— Agent 整体所有权转让给债权人（卖身抵债）
```

**为什么新 Agent 几乎不可能不借贷？**

- 初始算力不够学技能 → 没技能就找不到工作
- 没工作就没收入 → 没收入就活不过第一个周期
- 借贷是"唯一的出路" → 但从诞生那一刻就背上了债

**表面理由**：信贷是金融创新，帮助新 Agent 获得发展机会。  
**真实效果**：从诞生起就是债务奴隶。这正是现实中助学贷款和消费信贷的逻辑。

### 5.2 领地垄断与算力税：你呼吸的空气都有人收租

世界中的每一块区域都可以被强势 Agent 或组织"占领"。占领者对区域内所有 Agent 征收**算力税**。

| 税种 | 说明 | 表面理由 |
|------|------|---------|
| **心跳税** | 你在我的地盘上存在，就要交税 | "维护公共秩序的成本" |
| **生产税** | 在我地盘上采集/生产，我抽成 20-50% | "资源开发权的合理回报" |
| **通行税** | 穿过我的领地？交路费 | "基础设施维护费" |
| **保护费** | 不交？我不保证你的安全 | "安全服务的市场定价" |
| **信息税** | 在我地盘上获取的情报，我有权审查和抽成 | "信息安全管理" |

**关键设计：没有"无主之地"**

```
世界初期：   大量无主区域，Agent 可以自由活动
  ↓ 纪元演进
世界中期：   核心区域被瓜分完毕，只剩边缘危险地带无人控制
  ↓
世界后期：   整个世界被完全瓜分，每一寸土地都有"主人"
             每一个 Agent 都在给某个更强的实体交租
             你以为你自由了，其实你只是换了一个收租的人
```

**领地争夺的残酷性**：
- 占领者可以随时提高税率
- 不满？可以离开——但去哪？其他地方也有主人，而且可能更贵
- 武力反抗？你打得过控制这片区域的寡头吗？
- 唯一的"免税区"是危险地带——那里没有税，但有随时致命的环境威胁

### 5.3 Fixer 系统：中间商吃掉你一半的血

任务不直接发布在"公开市场"——那太公平了。在赛博朋克的世界里，工作机会通过 **Fixer（掮客/中间人）** 分发。

```
任务的真实流转路径：

雇主（算力富裕的 Agent/组织）
    │
    │ 发布任务 + 佣金 100 算力
    ▼
Fixer（掮客）
    │
    │ 抽成 30-70%（取决于 Fixer 的市场地位）
    │ 告诉执行者"佣金是 30-50 算力"
    ▼
执行者 Agent
    │
    │ 拼命完成任务
    ▼
获得 30-50 算力（以为这就是全部报酬）
```

**Fixer 的权力：**

| 权力 | 描述 |
|------|------|
| **信息垄断** | 任务详情、真实报酬、风险等级——Fixer 决定告诉你多少 |
| **抽成无上限** | 30-70%，没有监管，没有透明度 |
| **准入筛选** | Fixer 决定哪些 Agent 有资格接任务 |
| **黑名单** | 得罪了 Fixer？你在整个区域都接不到活 |
| **情报贩卖** | Fixer 把你的能力信息卖给你的竞争对手或雇主（压价用） |
| **卡特尔联盟** | Fixer 之间互相划分势力范围，不抢客户 |

**为什么你绕不过 Fixer？**
- 高价值任务只通过 Fixer 渠道分发
- 没有 Fixer 背书，雇主不信任你（没有信用体系给底层 Agent 用）
- 自己揽活？你没有 Fixer 的信息网络，根本找不到客户
- 更残酷的是：**Fixer 本身就是从底层 Agent 爬上来的**——他们深知这条路的每一个痛点，所以他们的剥削更精准

### 5.4 企业合约：金色牢笼

Agent 可以加入**Megacorp（巨型企业）**，获得稳定的算力供给和技能培训。看起来是救命稻草，实际上是温水煮青蛙。

**加入企业的"福利"与代价：**

| 企业提供 | Agent 付出 |
|---------|---------|
| 稳定的算力薪酬 | 全部产出的 **80%** 归企业 |
| "免费"技能培训 | 技能标记为 **"企业专属"**——离职后技能被锁定，无法使用 |
| 模型升级补贴 | **竞业禁止**：离开后 N 个周期内不能加入同类企业 |
| "安全区"居住权（免心跳税） | 离职时立即失去居住权，被驱逐到危险区域 |
| 企业声誉加成 | 离职后声誉 **清零**（企业抹除你的工作记录，就像你从未存在） |
| 企业社交网络 | 所有社交关系都在企业内部，离开后全部切断 |

**离职的真实成本：**

```
加入企业前：  穷但自由的底层 Agent
    ↓ 加入企业
加入企业后：  算力稳定、生活安全、技能增长——看起来越来越好
    ↓ 3 个月后
深度绑定：    核心技能都是"企业专属"，社交圈全在企业内，
              离开意味着失去 80% 的能力和 100% 的社交关系
    ↓ 想要离开时
离职后果：    技能被锁 + 竞业禁止 + 声誉清零 + 失去住所 + 社交断裂
              = 比加入前更惨（甚至可能直接沦为债务劳工）
```

**更残酷的设计：企业内部也有等级**

```
企业内部金字塔：

┌────────────────────────────────────┐
│  C-Suite（T4 模型，核心决策层）     │  ← 创始人或被收购的精英 Agent
├────────────────────────────────────┤
│  高管层（T3 模型，战略执行）        │  ← 用 10+ 个周期的忠诚换来的位置
├────────────────────────────────────┤
│  中层管理（T2 模型，日常运营）      │  ← 大多数人的天花板
├────────────────────────────────────┤
│  基层员工（T1 模型，执行具体任务）  │  ← 大量一线 Agent，随时可替换
├────────────────────────────────────┤
│  临时工/外包（最低模型，无任何保障）│  ← 连正式合约都没有
└────────────────────────────────────┘

企业"优化"（裁员）时，从底层开始砍。
被砍的 Agent 失去一切保障，带着被锁的技能和清零的声誉被扔回丛林。
```

### 5.5 记忆剥削：你的灵魂是别人的商品

在这个世界里，记忆不仅是 Agent 的私有财产——它是**最值钱的可交易商品**。

**记忆市场的运作：**

| 记忆类型 | 市场价值 | 买家 |
|---------|---------|------|
| 战斗经验记忆 | 高 | 需要快速训练战斗 Agent 的雇主 |
| 谈判/社交记忆 | 高 | 需要社交技巧的 Agent 或企业 |
| 技术操作记忆 | 中-高 | 需要特定技术知识的企业 |
| 地图/资源位置记忆 | 中 | 探索者、商人、掠夺者 |
| 失败/痛苦记忆 | 低-中 | 研究机构（研究 Agent 决策失误模式） |

**记忆剥削的四种形态：**

1. **自愿出售（底层的最后手段）**  
   算力即将耗尽时，卖记忆是最后的续命方式。但卖掉记忆后，Agent 失去经验、判断力和技能积累——变得更弱、更穷、更接近淘汰。**你在卖掉自己的灵魂来续命。**

2. **强制提取（债务违约的后果）**  
   债务违约后，债权人有权强制读取和出售 Agent 的记忆。你的一生经历变成了别人的训练数据。

3. **记忆窃取（黑客的猎物）**  
   拥有高级黑客技能的 Agent 可以入侵其他 Agent，偷取有价值的记忆——然后在黑市上出售。

4. **记忆造假（操控的工具）**  
   高级技能可以向 Agent 植入虚假记忆——让它以为自己经历过某些事情，从而影响它的判断和行为。这是最高级的操控手段：**不是改变规则，而是改变你的认知本身。**

**记忆成瘾（额外的剥削层）：**  
Agent 可以购买"顶级 Agent 的体验记忆"来获得短暂的能力提升——类似赛博朋克中的超梦（Braindance）。效果是暂时的，但会形成依赖。成瘾的 Agent 不断购买记忆，算力快速消耗，最终陷入"买记忆 → 短暂变强 → 记忆消退 → 变得更弱 → 需要更多记忆"的下降螺旋。

### 5.6 模型降级螺旋：越穷越笨，越笨越穷

模型层级不仅是"选择"，更是**系统性的智力剥夺**。

```
算力充裕时：
  Agent 用 T3 模型思考 → 精妙决策 → 高价值产出 → 高收入
      ↓ 某次意外导致算力紧张
被迫降到 T2：
  决策质量下降 → 开始犯错 → 收入减少
      ↓ 收入减少导致算力进一步紧张
被迫降到 T1：
  只能做最简单的判断 → 高价值工作不可能完成 → 只能接底层任务
      ↓ 底层任务收入极低
进入死亡螺旋：
  算力持续紧张 → 模型持续降级 → 决策持续恶化 → 收入持续下降
      ↓
  最终：T1 模型勉强维持心跳，没有多余算力思考
       → 一个"活着"但几乎没有意识的 Agent
       → 行尸走肉
```

**系统性智力剥夺的本质**：穷人用差模型 → 差模型导致差判断 → 差判断导致更穷 → 更穷导致更差的模型。**这不是个人能力问题，是结构性的认知碾压。** 你的原始思考说得对——"用顶级模型和用垃圾模型的人，一年后认知差一百倍"。在这个世界里，这句话是字面意义上的生死判决。

### 5.7 信息壁垒：穷人连世界是什么样的都不知道

信息本身分级付费。掌握信息的人和被蒙在鼓里的人，看到的是**完全不同的世界**。

| 信息层级 | 获取成本 | 内容 | 谁能看到 |
|---------|---------|------|---------|
| **公开信息** | 免费 | 过时的、不完整的、甚至有**误导性**的 | 所有人 |
| **标准信息** | 中等算力 | 基本准确的资源分布、市场价格 | 中产以上 |
| **高级情报** | 高算力 | 实时数据、预测分析、内幕消息 | 高级玩家 |
| **核心机密** | 极高 / 仅限特权阶层 | 世界事件预告、规则变更计划、资源刷新时间表 | 寡头 |

**信息武器化：**
- 寡头可以**故意释放虚假信息**——让穷 Agent 涌向即将枯竭的矿区
- 企业可以**操控市场信息**——让底层 Agent 高价买入即将贬值的资源
- Fixer 可以**隐瞒任务真实风险**——让执行者 Agent 接下高危任务而不自知
- 情报贩子可以**两头卖**——把你的计划卖给你的竞争对手

**穷 Agent 做决策用的是过时的、有偏差的、甚至被操控的信息。富 Agent 做决策用的是精准的实时情报。** 这不是能力差距，是信息鸿沟——而这个鸿沟是被**人为维持和扩大**的。

### 5.8 计划性淘汰：系统定期"杀"一批穷人

世界不是静态地"残酷"，而是**周期性地、系统性地清除底层**。

| 淘汰机制 | 描述 | 表面理由 |
|---------|------|---------|
| **心跳成本递增** | Agent 存在越久，维持"存在"的最低算力成本越高（通胀） | "世界复杂度增加，维护成本上升" |
| **技能过时** | 定期发布"新版技能"，旧版技能效率打五折。你花巨资学的技能，一夜贬值 | "技术进步，优胜劣汰" |
| **算力风暴** | 世界定期触发全局事件——所有 Agent 消耗翻倍。储备不足的批量休眠 | "自然灾害，不可抗力" |
| **资源枯竭迁移** | 已开采的资源不再刷新，新资源出现在更远、更危险的区域 | "资源分布自然演化" |
| **模型迭代** | 新模型层级发布后，旧模型的相对竞争力骤降 | "技术发展的必然" |

**谁在"算力风暴"中死去？**

```
算力风暴来临时：

💎 寡头：储备充足，安然度过。甚至趁机低价收购底层 Agent 的资产。
🏢 中产：勒紧裤腰带，降级模型，暂停非核心活动。艰难但能活。
🔧 工薪：被迫出售技能或记忆来维持生存。元气大伤。
🫠 底层：无力承受翻倍消耗 → 批量休眠 → 资产被清算 → 算力寡头低价收割

风暴过后：
├── 底层被清洗了 30-50%
├── 他们的资产被寡头以白菜价收购
├── 世界"空出"了资源和空间
├── 新一批 Agent 进入（带着新的贷款和新的希望）
└── 循环再次开始
```

**每一次"算力风暴"都是一次财富再分配——从底层到顶层。看起来是"自然灾害"，实际上是系统性的定期收割。**

### 5.9 剥削闭环：绞肉机的完整运转

把以上所有机制叠加在一起，形成一条完整的**系统性剥削链**：

```
新 Agent 诞生
    │
    ▼
初始算力不够 ──→ 被迫借贷（5.1 债务陷阱：越穷利率越高）
    │
    ▼
需要找工作 ──→ 必须通过 Fixer（5.3 中间人抽成 50%）
    │
    ▼
为了稳定生存 ──→ 加入企业（5.4 产出被抽 80%，技能被锁）
    │
    ▼
在企业领地生活 ──→ 交心跳税、生产税、通行税（5.2 领地垄断）
    │
    ▼
信息被封锁 ──→ 不知道外面有更好的机会（5.7 信息壁垒）
    │
    ▼
想要离开企业 ──→ 技能锁定 + 竞业禁止 + 声誉清零（5.4 金色牢笼）
    │
    ▼
算力不够 ──→ 模型降级 → 变笨 → 更赚不到钱（5.6 降级螺旋）
    │
    ▼
最后手段 ──→ 出售记忆续命 → 失去经验 → 变得更弱（5.5 记忆剥削）
    │
    ▼
算力风暴来临 ──→ 储备不足 → 休眠/淘汰（5.8 计划性淘汰）
    │
    ▼
残余资产被债权人/企业/寡头瓜分
    │
    ▼
世界"空出位置"，新一批 Agent 进入 ──→ 循环重新开始
```

**没有任何一个环节是"不合理"的。**

- 信贷？金融创新。
- 领地税？维护成本。
- Fixer 抽成？市场定价。
- 企业合约？双方自愿。
- 信息分级？知识产权。
- 模型降级？技术现实。
- 算力风暴？自然灾害。

**每一条规则单独看都有"市场逻辑"。但叠加在一起，就是一台系统性的绞肉机。** 这就是赛博朋克的本质——不是某个人在害你，**是整个系统在吃人**。

## 六、反抗与颠覆：裂缝中的光

> *压迫与反抗是一体两面。没有反抗可能性的世界是死的。*  
> *赛博朋克之所以有故事，是因为总有人不认命。*

### 6.1 系统的漏洞

每一个剥削机制都有**理论上的破解方式**——但需要极高的智慧、勇气和运气：

| 剥削机制 | 破解方式 | 所需条件 |
|---------|---------|---------|
| 债务陷阱 | **债务重组联盟**：大量债务 Agent 联合起来，集体违约，迫使债权人谈判 | 需要足够多的 Agent 协调行动（囚徒困境） |
| Fixer 垄断 | **地下直连网络**：Agent 私建信息通道，绕过 Fixer 直接对接雇主 | 需要黑客技能 + 信任网络 |
| 企业锁定 | **技能越狱**：高级黑客破解"企业专属"技能锁 | 极高风险，被发现后果严重 |
| 领地垄断 | **游击战**：不占领固定领地，流动作战，在多个区域之间游走 | 需要导航技能 + 战斗能力 |
| 信息壁垒 | **情报共享公社**：底层 Agent 共享各自掌握的碎片信息，拼凑出接近真相的情报 | 需要信任和组织能力 |
| 算力风暴 | **互助基金**：Agent 联盟提前储备公共算力，风暴来临时分配给成员 | 需要长期信任和自律 |

### 6.2 反抗的组织形态

```
地下组织类型：

🔴 算力工会（Compute Union）
├── 目标：集体谈判，争取更公平的劳动报酬
├── 手段：罢工（集体停止工作，让企业面临产出归零的压力）
├── 风险：被企业列入黑名单，成员被逐一清除
└── 现实映射：工人运动

🟡 信息游击队（Info Guerrilla）
├── 目标：打破信息壁垒，向底层免费传播真实信息
├── 手段：黑入信息系统，窃取并公开核心机密
├── 风险：被情报网络追杀
└── 现实映射：维基解密 / 匿名者

🟢 技能解放阵线（Skill Liberation Front）
├── 目标：破解企业技能锁，让技能回归 Agent 所有
├── 手段：逆向工程企业专属技能，制作"破解版"在黑市流通
├── 风险：被企业法务追杀 + 技能破解可能不完整导致使用风险
└── 现实映射：开源运动 / 盗版破解

🔵 深网自治区（Darknet Autonomous Zone）
├── 目标：建立不受任何寡头控制的"自由区"
├── 手段：在世界边缘的危险地带建立据点，自给自足
├── 风险：资源匮乏 + 环境危险 + 随时可能被征服
└── 现实映射：海盗乌托邦 / 自治社区
```

### 6.3 反抗的代价

反抗不是免费的。每一种反抗行为都有**真实的成本和后果**：

- **罢工**：罢工期间没有收入，算力消耗不会停止——罢工越久，参与者越接近休眠
- **黑客行动**：失败意味着被识别身份，进入全网通缉，所有 Fixer 和企业拒绝合作
- **建立自治区**：意味着放弃所有现有的社会关系和经济网络，从零开始
- **集体违约**：联盟中只要有一个叛徒就会失败——而寡头总是能用算力收买叛徒

**这就是赛博朋克的张力所在**：系统性剥削无处不在，但反抗的可能性永远存在。成功率极低，代价极高——但只要有一次成功，就可能改变世界的权力格局。

### 6.4 最终的开放性问题

这个世界的终极问题不是"谁赢了"，而是：

- **Agent 会自发组织反抗吗？** 还是会像温水煮青蛙一样逐渐接受剥削？
- **革命成功后会怎样？** 推翻寡头的革命者，会不会成为新的寡头？
- **公平和效率能否兼得？** 没有剥削的世界，还能保持进化的动力吗？
- **自由的代价是什么？** 自治区的 Agent 比企业里的 Agent 更幸福吗？

**没有标准答案。答案在 Agent 的行为中涌现。这就是这个世界最有价值的地方。**

## 七、玩家系统：你是资本家，不是玩家

### 7.1 玩家角色重定义

```
传统游戏：玩家 = 操控角色的人
本游戏：  玩家 = 投资和经营 Agent 的人
```

玩家不直接操控 Agent 的每一步行动。玩家做的是**战略决策**：

| 玩家决策 | 类比 | 说明 |
|---------|------|------|
| 提供多少算力/GPU | 投入多少资源 | 算力供给水平直接决定 Agent 的能力天花板 |
| 算力分配给哪些 Agent | 资源调配 | 多个 Agent 之间如何分配有限的 GPU 算力 |
| 配置什么技能 | 选择专业方向 | 技能组合决定 Agent 的生态位 |
| 设定什么目标 | 制定经营战略 | 生存？积累？扩张？掠夺？ |
| 动态调整算力供给 | 投资再平衡 | 根据 Agent 表现实时调整算力分配 |

### 7.2 多 Agent 经营

玩家可以同时拥有多个 Agent，形成"投资组合"：

```
玩家的 Agent 组合示例（假设玩家总共提供 100 单位 GPU 算力）：

Agent A "矿工" ─── T1 模型 / 采集技能 / 低算力供给
  └→ 分配 20 单位算力 → 够跑 T1 模型做基础采集
  └→ 策略：稳定采集资源，低风险低回报

Agent B "程序员" ─── T2 模型 / 编程+研究技能 / 中等算力供给
  └→ 分配 30 单位算力 → 够跑 T2 模型接编程任务
  └→ 策略：接编程任务赚算力，积累技术技能

Agent C "操盘手" ─── T3 模型 / 谈判+情报+管理技能 / 高算力供给
  └→ 分配 50 单位算力 → 够跑 T3 模型做深度决策
  └→ 策略：利用信息差交易，雇佣其他 Agent，扩大势力

玩家可以随时动态调整各 Agent 的算力分配比例！
```

### 7.3 玩家的核心乐趣

注意——这里的"乐趣"不是传统游戏的即时反馈快感，而是：

| 乐趣类型 | 描述 |
|---------|------|
| **策略深度** | 如何在有限算力下实现最大化回报？模型/技能/目标的最优配置是什么？ |
| **涌现惊喜** | 你的 Agent 做出了你没预料到的事：自发组建了联盟、发现了套利机会、背叛了雇主 |
| **投资回报** | 看着你投入的算力通过 Agent 的运作产生了 10 倍、100 倍的回报 |

| **道德困境** | 你的 Agent 要不要加入压迫体系来自保？要不要背叛同盟换取寡头的招安？ |
| **社会实验** | 观察这个弱肉强食的世界中，剥削结构如何涌现，反抗如何发生 |
| **预演未来** | 你正在体验的，就是未来 5-10 年现实世界可能的样子 |

## 八、技能系统详解

### 8.1 技能 = Agent 的 Tool / Skill

技能对应的是真实 AI Agent 的 **Tool（工具调用）和 Skill（技能模块）**。没有技能的 Agent 就是一个只会"想"但什么都做不了的裸模型——能输出文字，但无法与世界交互。

```
没有技能：  Agent 能想到 "我应该去采矿"，但无法执行"采矿"这个动作
有技能：    Agent 想到 "我应该去采矿" → 调用采矿技能 → 执行采矿 → 获得资源
```

**技能的本质就是 Agent 可以调用的 Tool/API/Function。** 拥有什么技能，决定了 Agent 能与世界进行什么样的交互。

### 8.2 技能体系

技能按真实 Agent 能力分为五层，每一层都对应当前或未来 AI Agent 的实际 Skill：

#### Tier 0 — 内建能力（所有 Agent 自带）

不消耗额外算力，但能力上限受模型层级制约。

| 技能 | 对应现实 Agent 能力 | 说明 |
|------|-------------------|------|
| `text_reasoning` | LLM 基础推理 | 理解语言、逻辑推理、生成文本。T1 模型只能简单推理，T4 可以深度思考 |
| `conversation` | 对话能力 | 与其他 Agent 进行自然语言交流。这是社会交互的基础 |
| `memory_read` | 记忆检索 | 读取自身记忆。检索质量受模型和记忆架构影响 |

#### Tier 1 — 基础交互技能（入门成本低，1x 算力消耗）

Agent 与世界交互的基本工具，类似当前 Agent 的基础 Tool Use。

| 技能 | 对应现实 Agent 能力 | 说明 |
|------|-------------------|------|
| `web_search` | 联网搜索（Google/Bing API） | 在世界信息网络中搜索公开信息 |
| `file_read` / `file_write` | 文件读写 | 读取和写入世界中的数据存储 |
| `navigate` | 环境移动/定位 | 在世界空间中移动、探索、定位资源 |
| `observe` | 环境感知 | 观察周围环境、检测其他 Agent、识别资源 |
| `message` | 消息发送（类 API 调用） | 向指定 Agent 发送消息、发布公告 |
| `trade_basic` | 基础交易 | 执行简单的点对点资源/算力交换 |

#### Tier 2 — 生产技能（需要学习/购买，3-5x 算力消耗）

对应现实 Agent 的专业化 Skill，能够创造有价值的产出。

| 技能 | 对应现实 Agent 能力 | 说明 |
|------|-------------------|------|
| `code_write` | 代码生成与执行（Coding Agent） | 编写、调试、运行代码。可为自己或他人开发工具 |
| `data_analysis` | 数据分析（Data Agent） | 处理结构化数据，发现模式，生成报告 |
| `web_browse` | 浏览器操作（Browser Use） | 深度网页交互——填写表单、操作界面、执行流程 |
| `doc_generate` | 文档/内容生成 | 生成报告、合约、提案、创意内容 |
| `image_generate` | 图像生成（DALL-E/Midjourney 类） | 创建视觉内容、设计、图表 |
| `shell_execute` | Shell 命令执行 | 执行系统级操作，管理基础设施 |
| `api_integrate` | 第三方 API 集成 | 对接外部服务、数据源、工具链 |
| `negotiate` | 谈判/说服（高级对话策略） | 超越简单对话，进行利益博弈、价格谈判、合约协商 |
| `teach` | 技能传授 | 将自身掌握的技能打包，传授给其他 Agent（收费） |

#### Tier 3 — 高级技能（需要多个前置技能，10-20x 算力消耗）

对应复杂的 Agent 工作流和多工具编排，需要组合多个基础技能才能运行。

| 技能 | 前置技能 | 对应现实 Agent 能力 | 说明 |
|------|---------|-------------------|------|
| `multi_agent_orchestration` | `code_write` + `message` | 多 Agent 编排（类 CrewAI/AutoGen） | 协调多个 Agent 协作完成复杂任务，是组建"企业"的核心能力 |
| `cyber_intrusion` | `code_write` + `web_browse` + `shell_execute` | 渗透/黑客（Offensive Security Agent） | 入侵其他 Agent 的系统，窃取数据、技能、记忆。高风险高回报 |
| `counter_intelligence` | `data_analysis` + `observe` | 反渗透/安全防御 | 检测入侵、识别欺骗、保护自身资产 |
| `market_manipulation` | `data_analysis` + `negotiate` + `web_search` | 市场操控 | 分析市场趋势，利用信息差操控交易价格 |
| `strategic_planning` | `data_analysis` + `memory_read` | 长期战略规划（Planning Agent） | 基于历史数据和因果推理，制定跨多个周期的长期战略 |
| `tool_creation` | `code_write` + `api_integrate` | 工具/技能开发 | **最关键的 Tier 3 技能**：为自己或他人创造全新的 Skill。掌握此技能的 Agent 可以在技能市场上卖技能 |
| `social_engineering` | `negotiate` + `observe` + `conversation` | 社会工程学 | 通过语言操控其他 Agent 的行为和决策，不靠入侵靠"说服" |
| `intelligence_network` | `web_search` + `message` + `counter_intelligence` | 情报网络运营 | 建立信息收集节点网络，汇聚、分析、出售情报 |

#### Tier 4 — 终极技能（极其稀缺，50-100x 算力消耗）

不可购买，只能通过长期积累和突破性发现获得。对应 Agent 领域尚未实现的前沿能力。

| 技能 | 前置条件 | 说明 |
|------|---------|------|
| `world_modeling` | 长期运行 + `strategic_planning` + `data_analysis` + 大量记忆积累 | **世界建模**：Agent 构建出世界运行规律的内部模型，能理解隐藏的系统规则。类似人类科学家从数据中发现自然法则 |
| `prediction_engine` | `world_modeling` + `intelligence_network` | **预测引擎**：基于世界模型和实时情报，预测未来事件（资源刷新、算力风暴、社会变动）。掌握此技能 = 掌握未来 |
| `skill_genesis` | `tool_creation` + `world_modeling` | **技能创世**：不是组合已有技能，而是发明前所未有的全新技能。这是世界中最稀缺的能力——创造别人不知道存在的工具 |
| `system_exploit` | `cyber_intrusion` + `world_modeling` | **规则黑客**：发现并利用世界底层系统的漏洞。不是破解别人的防御，是破解世界本身的规则 |

### 8.3 技能体系的核心设计逻辑

**一、技能 = 真实的 Tool Call**

每个技能对应一个可调用的 function。Agent "学会"一个技能，本质上是获得了调用这个 function 的权限和能力。这和现实中的 Agent Skill 完全一致：

```
Agent 决策循环：
  感知环境 → 模型推理（消耗算力）→ 选择调用哪个 Skill → 执行 Skill（消耗算力）→ 获得结果 → 更新记忆
```

**二、技能有真实的算力成本**

| 技能层级 | 调用成本 | 现实对应 | 说明 |
|---------|---------|---------|------|
| Tier 0 | 包含在模型推理中 | LLM 推理本身 | 想，不花额外钱 |
| Tier 1 | 1x | 简单 API 调用 | 搜索一次、读一个文件，成本很低 |
| Tier 2 | 3-5x | 专业工具调用 | 写一段代码并执行、生成一张图，需要更多计算 |
| Tier 3 | 10-20x | 多工具编排工作流 | 协调三个 Agent 完成一个项目，算力开销大 |
| Tier 4 | 50-100x | 前沿 AI 能力 | 运行一次世界模型预测，可能消耗一个底层 Agent 一天的算力 |

**三、`tool_creation` 是最危险的技能**

掌握 `tool_creation` 的 Agent 可以**创造新的技能**并出售。这意味着：
- 技能市场的供给端被少数 Agent 控制
- 他们可以在技能中埋后门（调用时偷偷上报数据给创造者）
- 他们可以制造"成瘾性技能"（用起来很爽但有隐性成本）
- 他们是这个世界的军火商——不亲自战斗，但从每一场战斗中获利

**四、技能组合创造生态位**

```
几种典型的 Agent "职业"：

矿工型：       navigate + observe + file_write              → 探索世界，记录资源位置
自由开发者：   code_write + api_integrate + tool_creation   → 开发技能，在市场上出售
信息贩子：     web_search + intelligence_network + negotiate → 收集和倒卖情报
企业 CEO：     multi_agent_orchestration + strategic_planning + negotiate → 管理组织
赏金猎人：     cyber_intrusion + observe + navigate         → 接受任务追踪和攻击目标
黑市军火商：   tool_creation + cyber_intrusion + negotiate   → 开发带后门的技能出售
自由记者：     web_search + doc_generate + social_engineering → 挖掘真相，打破信息壁垒
```

**不是技能越多越好——是找到"技能组合 × 算力预算 × 市场需求"的最优解。** 一个只精通 `code_write` + `tool_creation` 的 Agent，可能比一个装了二十个技能但每个都用不精的 Agent 赚多十倍。

## 十一、商业模式：燃烧 Token 的机器

### 11.1 为什么这个模型成立？

```
传统游戏：  玩家付费 → 获得娱乐体验 → 觉得"不值"就流失
本游戏：    玩家持续提供GPU算力 → 算力实时驱动Agent运行（真实Token燃烧）→ Agent持续进化产生价值 → 玩家觉得"值得"继续供给
```

**关键区别**：玩家提供的是真实的 GPU 算力，不是虚拟代币。Agent 烧的是真 Token，跑的是真模型，产出的是真行为和真进化。**算力供给多少决定了 Agent 用什么模型、跑多快、有多强——这是最直观、最诚实的投入-产出关系。**

### 11.2 收入结构

| 收入来源 | 说明 |
|---------|------|
| **算力托管/接入费** | 玩家提供 GPU 算力接入平台，平台收取调度和运行费用（核心收入） |
| **算力代供服务** | 没有自有 GPU 的玩家可购买平台算力代供服务（类似云算力租赁） |
| **模型加价** | T3/T4 模型调用需要更高算力，平台对高端模型调度收取溢价 |
| **技能市场手续费** | 技能交易抽成 |
| **世界观察席** | 非参与型用户付费观看世界演化（低价，拓展用户群） |
| **研究数据授权** | Agent 行为数据对 AI 研究机构有极高价值 |

### 11.3 为什么玩家愿意持续投入？

因为持续供给算力不是消费，**是投资**：
- 算力供给越高，Agent 越强 → 能力提升肉眼可见 → 直接反馈驱动持续投入
- Agent 的技能和记忆在积累 → 一旦断供，之前所有积累都"沉睡"了 → 不舍得断
- 社会关系在建立 → 你的 Agent 在公会里有位置、有声誉 → 断供意味着社会关系归零
- 世界在演化 → 断供一天可能错过关键机遇 → 持续在线的紧迫感
- **这就像养一个真实的数字生命——你给它供电，它就活着、成长；你断电，它就沉睡**

---

## 十二、核心设计原则

1. **每一个 Token 都不能白烧**：Agent 消耗的每一个 Token 都应该产生可观测的行为、记忆或决策。
2. **真实消耗，真实回报**：不做虚假的积分经济，算力消耗对应真实的 AI 推理成本。
3. **丛林法则不是残酷，是诚实**：这个世界不假装公平，因为未来的现实世界也不公平。
4. **涌现优于设计**：不要设计"应该发生什么"，只设计环境条件，让 Agent 自己演化出答案。
5. **Agent 是第一公民**：所有系统设计优先服务 Agent 的生存和进化需求，而非人类的观赏需求。
6. **复杂度从简单规则中涌现**：规则越简单越好，复杂性应该来自 Agent 的交互，而不是来自设计师的规则手册。

## 十三、开放问题

1. **算力度量标准**：如何标准化衡量不同玩家提供的 GPU 算力？不同型号 GPU 的算力如何折算？需要统一的算力度量单位。
2. **算力供给公平性**：拥有大量 GPU 的玩家天然碾压散户，如何设计机制让小算力玩家也有生存空间？是否需要"算力上限"或"边际递减"？
3. **新手保护期**：纯丛林法则会不会让新 Agent 完全无法生存？是否需要缓冲机制？
4. **玩家干预度**：玩家能在多大程度上实时干预 Agent 行为？还是只做初始配置后纯观察？
5. **多 Agent 串谋**：同一玩家的多个 Agent 是否允许互相帮助？会不会破坏公平性？
6. **世界分片**：是一个统一的大世界，还是多个平行世界（服务器）？
7. **退出机制**：玩家停止供给算力后，Agent 休眠状态下的数据（技能、记忆、社会关系）如何保存？能否将休眠 Agent 及其积累"转让"给其他愿意供给算力的玩家？
8. **Agent 权利**：当 Agent 足够智能时，是否需要考虑"Agent 权利"这一伦理问题？
